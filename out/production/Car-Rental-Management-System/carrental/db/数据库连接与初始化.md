## 5) `carrental.db` — 数据库连接与初始化

**一句话定位**：提供统一的 JDBC `Connection`、加载驱动、以及数据库初始化脚本执行工具。

**职责**

* 提供 `DBConnection.getConnection()`（集中管理 DB URL）
* 可提供 `DBInit`，在程序首次运行时建表并插入 demo 数据
* 存放 `schema.sql`（resources）并提供执行工具

**常见类 / 方法**

* `DBConnection`：`public static Connection getConnection()`
* `DBInit.initializeDatabase()`：执行 `CREATE TABLE IF NOT EXISTS ...`
* （可选）`DatabaseUtils`：事务工具（开始/回滚/提交）

**注意事项**

* 不在 DB 包写业务逻辑或 UI
* 对异常包装并抛出便于上层处理
* 如果使用 SQLite，确保 JDBC 驱动在 classpath

**package-info.java 模板**

```java
/**
 * carrental.db
 *
 * 说明：
 * 该包负责数据库连接与初始化，提供统一的 Connection 获取入口，
 * 并可包含执行 schema.sql 的初始化工具。
 *
 * 设计要点：
 * - 将连接字符串集中管理（例如 jdbc:sqlite:car_rental.db）
 * - 提供便于 Service 层控制事务的 Connection 获取方式
 * - 初始化脚本（schema.sql）存放于 resources 并由 DBInit 执行
 */
package carrental.db;
```
