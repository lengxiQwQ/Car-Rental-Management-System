## 4) `carrental.ui` — 用户界面（Swing 界面）

**一句话定位**：所有 Swing 窗体/面板/对话框代码，负责接受用户输入并调用 Service 层。

**职责**

* 提供登录页面、仪表盘、车辆管理表格、租车/还车弹窗、报表界面等
* 只调用 Service 层来执行操作，不直接访问 DAO
* 在事件处理（ActionListener）中使用 `SwingWorker` 执行耗时操作（避免阻塞 EDT）

**常见类 / 组件**

* `Main`（程序入口，初始化 DB 并显示 LoginFrame）
* `LoginFrame`（用户名/密码 -> 调用 UserService.login）
* `AdminDashboard`, `StaffDashboard`（不同角色视图）
* `CarManagementPanel`（JTable + CRUD 按钮）
* `RentalFormDialog`, `ReturnFormDialog`
* `ReportPanel`（表格/导出 CSV）

**注意事项**

* 所有长时间任务（数据库查询、报表生成）必须放后台线程（`SwingWorker`），否则界面卡死
* 只捕获并显示用户友好的错误信息；将详细异常写日志
* 避免在 UI 层写复杂逻辑，保持薄表示层

**package-info.java 模板**

```java
/**
 * carrental.ui
 *
 * 说明：
 * UI 层包含所有 Swing（或 JavaFX）窗口、面板与对话框。UI 负责接收用户交互，
 * 校验基本输入并调用 service 层接口来完成具体操作。
 *
 * 设计要点：
 * - 避免在 EDT（事件分发线程）中执行长时间数据库或计算操作，使用 SwingWorker
 * - UI 只调用 service，不直接访问 DAO
 * - 对用户显示友好的错误信息，同时将详细错误记录到日志
 *
 * 常见类：
 * - Main, LoginFrame, AdminDashboard, StaffDashboard, CarManagementPanel, RentalFormDialog
 */
package carrental.ui;
```
